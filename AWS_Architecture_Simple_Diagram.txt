DATA FLOW (Training Pipeline)
==============================

1. [Clinician] → Uploads de-identified MRI → [S3 Bucket]
                                              ↓
2. [Data Scientist] → Triggers training → [SageMaker/EC2]
                                              ↓
                                    Trains nnU-Net model
                                              ↓
3. Model outputs → Segmentation masks → Saved to [S3]
                                              ↓
4. Performance metrics → Logged to → [RDS/DynamoDB]
                                              ↓
5. [Frontend Demo] ← Fetches results ← [API Gateway]


KEY SECURITY FEATURES
=====================

✓ All data de-identified (no PHI)
✓ S3 encryption at rest (AES-256)
✓ HTTPS/TLS for data in transit
✓ IAM role-based access control
✓ VPC isolation for compute resources
✓ CloudTrail audit logging
✓ MFA enforcement for admin access


WHY WE NEED EACH COMPONENT
===========================

S3:          Store 40GB of medical images securely
EC2/SageMaker: Train deep learning models (requires GPU)
Lambda/API:   Connect frontend to backend without managing servers
RDS:          Track experiments and model versions (reproducibility)
IAM:          Control who accesses what (HIPAA compliance)
